# iOS 替代方案

## 结论

截至 2026-04-10，iPhone 非越狱环境下，无法像 Android 一样拦截并按联系人或群聊粒度自动筛选微信通知。

原因在于 iOS 不向第三方 App 暴露微信通知内容读取与取消能力，通知扩展只能处理自己 App 发出的通知，不能改写微信这类第三方应用的通知。

## 可行替代

### 方案 A：专注模式 + 微信整体通知

- 建立“工作”和“非工作”两个专注模式。
- 工作模式允许微信通知。
- 非工作模式关闭微信通知或仅允许少量联系人来电提醒。

优点：
- 完全基于系统能力，稳定。
- 用户容易理解和配置。

缺点：
- 只能控制微信整体，不能精确到某个群或联系人。

### 方案 B：微信内手动置顶/免打扰配合专注模式

- 在微信里把重要联系人、重要群置顶。
- 普通群开启免打扰。
- 再配合 iOS 专注模式按时段切换。

优点：
- 不需要越狱。
- 能在一定程度上减少噪音。

缺点：
- 仍然无法由第三方 App 自动切换某个指定群的通知状态。
- 需要用户在微信内维护免打扰设置。

### 方案 C：企业侧替代通知通道

如果真实需求是“工作时间接收特定人/群的工作消息，非工作时间屏蔽”，可考虑把关键提醒迁移到企业自有 App、企业微信机器人、邮件或日历提醒，再由 iOS 原生通知系统精细控制。

优点：
- 可完全自动化。
- 可审计、可配置。

缺点：
- 不再是直接控制微信通知，而是改业务通知链路。

## 产品建议

如果要做双端产品，建议路线是：

1. Android：做真正的通知筛选器，支持多应用、多关键字、多时间段。
2. iOS：做“规则配置端 + Focus 配置向导 + 替代方案说明”，而不是承诺对微信通知做同等控制。

这样可以保证 Android 真能落地，iOS 也有合理体验预期。
